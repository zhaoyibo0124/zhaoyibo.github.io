git

Posted on 2019-12-17 | In 代码管理工具

git是一个开源的分布式版本控制系统，可以有效、高速地处理从很小到非常大的项目版本管理。Git 是 Linus Torvalds 为了帮助管理 Linux 内核开发而开发的一个开放源码的版本控制软件

git基本命令
以下命令基于git安装完成之后，如果未安装可以进行命令安装或去官网安装，安装步骤在此不做赘述；
git config –global user.name “xxxx”：配置用户名；
git config –global user.email “123@gmail.com“：配置账户；
git config -l:查看配置；
git init：初始化创建一个git仓库，创建之后就会在当前目录生成一个.git的文件；
git add [filename]：添加文件把文件添加到缓冲区；
git add . ：添加所有文件到缓冲区；
git add –all：加all 可以添加被手动删除的文件 二者的区别 加all可以添加被手动删除的文件；
git rm [filename]：删除文件；
git commit -m “提交的说明”: 提交缓冲区的所有修改到仓库(注意：如果修改了文件但是没有add到缓冲区，也是不会被提交的)commit可以一次提交缓冲区的所有文件；
git status：查看状态；
git diff [filename] ：比较提交的文件差异
git log：查看日志；
git reset:版本回退 git log 可以查看到提交的版本号;
–git reset –hard d7b5： 回退到指定版本;
–git reset –hard HEAD^： 回退到上一版本;
git reflog：查看命令历史;
git branch：查看分支;
git branch [branchName]:创建分支;
git checkout [branchName]:切换分支;
git checkout -b 分支名:创建分支并切换到创建的分支;
git merge 分支名:合并某分支的内容到当前分支;
git branch -d 分支名:删除分支;
git pull:拉取远程代码；
git push：提交代码；
git stash：保存当前进度 （如果当前分支所做的修改你还不想提交，但又需要切换到其他分支去查看，就可以使用该命令）；
.gitignore：忽略那些没必要的提交；

感谢您的支持
